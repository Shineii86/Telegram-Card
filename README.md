## Why This Exists

Sharing a Telegram link on GitHub, Twitter, or a blog renders as a dead URL — no preview, no stats, no personality. **Telegram Card Widget** solves this with a single URL that produces a polished, auto-updating card with the entity's avatar, name, type, verified badge, and live statistics.

## Quick Start

### Just Use the URL

```
https://telegramcard.vercel.app/?username=YOUR_TELEGRAM_USERNAME
```

Replace `YOUR_TELEGRAM_USERNAME` with any public Telegram username (without `@`). Done.

### GitHub README

```markdown
[![Telegram](https://telegramcard.vercel.app/?username=Shineii86&theme=dark)](https://t.me/Shineii86)
```

### With Custom Colors

```
https://telegramcard.vercel.app/?username=Shineii86&bgColor=rgba(15,23,42,1)&textColor=%23F8FAFC&extraColor=%2338BDF8
```

### With Custom Photo

```
https://telegramcard.vercel.app/?username=Shineii86&photo=https://example.com/avatar.png
```

## API Reference

### Endpoint

```
GET https://telegramcard.vercel.app/
```

### Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|:--------:|---------|-------------|
| `username` | `string` | ✅ | — | Telegram username (without `@`). `[a-zA-Z0-9_]` only. |
| `theme` | `string` | ❌ | `light` | Preset: `light` or `dark` |
| `bgColor` | `string` | ❌ | *theme* | Card background — any CSS color |
| `textColor` | `string` | ❌ | *theme* | Primary text (name, title) |
| `subtleTextColor` | `string` | ❌ | *theme* | Secondary text (@username) |
| `extraColor` | `string` | ❌ | *theme* | Accent (subscriber count, stats) |
| `shadowColor` | `string` | ❌ | *theme* | Card drop shadow |
| `fontFamily` | `string` | ❌ | `Inter, sans-serif` | CSS font-family stack |
| `verified` | `string` | ❌ | `auto` | `true` = force show, `false` = force hide, `auto` = detect |
| `photo` | `string` | ❌ | — | Custom avatar URL (must be `http`/`https`) |

### Color Formats

URL-encode `#` as `%23`. All standard CSS color values work:

| Format | Example | URL-Encoded |
|--------|---------|-------------|
| Hex | `#F1F5F9` | `%23F1F5F9` |
| RGB/RGBA | `rgba(30,41,59,0.8)` | `rgba(30,41,59,0.8)` |
| HSL | `hsl(215,25%,17%)` | `hsl(215,25%,17%)` |
| Named | `coral`, `steelblue` | as-is |

### Error Handling

The API always returns HTTP `200` — even errors produce a valid PNG with a styled error card. Your README will never show a broken `<img>` tag.

## Features

| ⚡ Performance | 🎨 Customization | 🔍 Intelligence |
|:---:|:---:|:---:|
| Sub-100ms edge response | 6 independent color overrides | Auto-detects User/Bot/Channel/Group |
| < 10ms cache hits | Light & dark presets | Scrapes live stats |
| 2-hour CDN cache | Custom fonts | Verified badge detection |
| ~15-40 KB images | 12 ready-made palettes | Theme-aware error cards |

| 🛡️ Security & Privacy | 🌍 Deployment | 🧩 Integration |
|:---:|:---:|:---:|
| Zero data storage | Vercel (recommended) | GitHub READMEs |
| Input sanitization | Cloudflare Workers | Discord embeds |
| CSP + nosniff headers | Netlify, Railway, Render | Notion, blogs, portfolios |
| No tracking or cookies | Any Next.js platform | Node.js, Python, Go, cURL |

## Technology Stack

| Layer | Technology | Why |
|-------|-----------|-----|
| Framework | [Next.js 15](https://nextjs.org/) | Edge runtime, App Router, built-in OG support |
| Language | [TypeScript 5](https://www.typescriptlang.org/) | Type safety, catches bugs at compile time |
| Rendering | [Satori](https://github.com/vercel/satori) + [@vercel/og](https://vercel.com/docs/concepts/functions/edge-functions/og-image-generation) | JSX → SVG → PNG at the edge — no headless browser |
| Scraping | [JSDOM](https://github.com/jsdom/jsdom) | Server-side DOM parsing of `t.me` pages |
| Infrastructure | [Vercel Edge Network](https://vercel.com/) | 100+ PoPs, zero-config edge functions |
